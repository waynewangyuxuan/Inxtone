3. **03_plot/foreshadowing.md** — 如有埋设或回收伏笔，更新状态
4. **_STATUS.md** — 更新进度

## 二、章节摘要日志

### 2.1 日志格式

```markdown
# 章节摘要日志

> AI提示：每章完成后更新此日志。格式固定，便于检索。

| 章节 | 标题 | 主要事件 | 出场人物 | 伏笔操作 | 字数 |
|------|------|----------|----------|----------|------|
| 001 | [标题] | [1-2句话概括] | C001,C002 | 埋FS001 | 3200 |
| 002 | [标题] | [1-2句话概括] | C001 | - | 2800 |
| 003 | [标题] | [1-2句话概括] | C001,C002,C003 | 埋FS002,收FS001 | 3500 |
```

### 2.2 摘要写作要点

- **简洁**: 1-2句话概括核心事件
- **客观**: 写"发生了什么"而非"感受"
- **完整**: 包含关键剧情转折
- **便于检索**: AI可以快速理解前情

## 三、写作技巧（来自B20）

### 3.1 对话写作（来自B20 5.1）

| 原则 | 说明 |
|------|------|
| **功能性** | 每段对话都应推动剧情或揭示人物 |
| **区分度** | 不同人物说话方式应有区别 |
| **潜台词** | 高水平对话有言外之意 |
| **节奏感** | 长短句交替，留白与密集交替 |

### 3.2 情感节奏（来自B20 5.2）

| 情感类型 | 写法要点 |
|----------|----------|
| 愤怒 | 短句、急促、动作描写 |
| 悲伤 | 长句、细节、内心独白 |
| 紧张 | 短句、省略、时间紧迫感 |
| 喜悦 | 活泼、动作、环境呼应 |
| 平静 | 长句、环境描写、舒缓节奏 |

### 3.3 场景描写锚点

使用预先设计的地点锚点（见01_world/locations.md），保持描写一致性。

## 四、Wayne原则在写作中的持续检查

### 4.1 每10章自检

每写完10章，进行一次Wayne原则自检：

```markdown
## 第 X-X 章 Wayne原则自检

### R1 "养狗"检查
- 这10章中，配角是否有自己的行动？[是/否]
- 配角是否有反对/质疑主角的时刻？[是/否]
- 问题章节: [列出]
- 改进措施: [描述]

### R2 符号化检查
- 配角是否只展示了单一面向？[是/否]
- 是否有角色可以用一个词概括？[是/否]
- 问题章节: [列出]
- 改进措施: [描述]

### R3 草菅人命检查
- 是否有无意义的死亡？[是/否]
- 死亡是否有余波？[是/否]
- 问题章节: [列出]
- 改进措施: [描述]

### P1-P4 核心原则检查
- 是否尊重了每个出场角色？[是/否]
- 配角是否有自己的想法？[是/否]
- 强弱对话是否有人格平等？[是/否]

### 总评
[这10章整体表现，需要注意的问题]
```

### 4.2 常见写作陷阱

| 陷阱 | 表现 | 解决方案 |
|------|------|----------|
| 配角工具化 | 配角只在主角需要时出现 | 给配角独立戏份 |
| 对话单一化 | 配角只会"是"、"好" | 让配角有不同意见 |
| 升级机械化 | 打怪→升级→打怪 | 加入人物内心变化 |
| 爽感堆砌 | 连续打脸无节奏 | 加入低谷/思考 |
| 世界观信息dump | 大段设定说明 | 融入剧情揭示 |

## 八、常见问题解答

### Q1: 写到一半发现大纲有问题怎么办？

**A**:
1. 小问题：直接调整，更新大纲
2. 大问题：暂停写作，回到Phase 4重新规划
3. 记录问题原因，避免下次重蹈覆辙

### Q2: 写不出来怎么办？

**A**:
1. 检查是否Context不足（缺少灵感）
2. 检查大纲是否太僵硬（给自己留空间）
3. 先写能写的部分，跳过卡住的地方
4. 与AI讨论可能的发展方向

### Q3: 读者反馈和原计划冲突怎么办？

**A**:
1. 小调整：顺应读者，更新大纲
2. 大调整：评估是否真的需要改，避免"被读者牵着走"
3. 核心原则（Wayne原则）不能因读者反馈而违背

### Q4: 如何保持长期写作的一致性？

**A**:
1. 依赖索引系统（_INDEX.md）
2. 定期重读章节日志（_chapter_log.md）
3. 每卷结束做一次全面回顾
4. AI辅助一致性检查
