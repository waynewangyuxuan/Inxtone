## 一、项目启动

### 1.1 新项目初始化

```
我正在使用网文创作Playbook开始一个新项目。

请阅读以下文件了解框架：
- PLAYBOOK_INDEX.md（总览）
- _guides/00_meta_guide.md（第一阶段指南）

现在请帮我完成 00_meta/concept.md 的填写。

我的初步想法是：
[描述你的创意：主角是谁？大概要做什么？有什么独特的点？]
```

### 1.2 阶段推进

```
当前项目状态见 _STATUS.md。

我已完成 Phase [X]，准备进入 Phase [X+1]。

请阅读 _guides/[对应guide].md 并帮我开始这个阶段的工作。

当前已完成的内容：
[列出已完成的关键文件]
```

## 二、构思阶段 (Phase 1)

### 2.1 创意讨论

```
我想写一本网文，初步想法是：
[你的创意]

请帮我用以下框架分析这个创意：
1. 一句话概括
2. 目标读者画像
3. 三个核心卖点
4. 成功模式归类（世界观驱动/人物驱动/爽点驱动/反套路驱动）
5. 潜在风险
```

### 2.2 Wayne原则红线检查

```
请阅读 00_meta/guardrails.md 中的Wayne原则。

现在检查我的创意是否有违反红线的风险：
[描述你的创意]

重点检查：
- 是否有"养狗式"人物关系倾向（R1）
- 主角崛起是否依赖工具人（R2）
- 世界观是否支持平等交往（R4）
```

### 2.3 差异化定位

```
以下是我的参考作品和我想借鉴/避免的点：
[参考作品1]: 借鉴[...], 避免[...]
[参考作品2]: 借鉴[...], 避免[...]

请帮我分析：
1. 我的创意与这些作品的最大区别是什么？
2. 读者为什么要选择我的书而不是重看这些经典？
3. 这个差异化定位有什么风险？
```

## 三、世界观阶段 (Phase 2)

### 3.1 力量体系设计

```
请阅读 _guides/01_world_guide.md 中关于力量体系的部分。

帮我设计一个力量体系，基于以下创意：
[你的力量体系初步想法]

要求：
1. 检查是否符合Wayne原则W1（支持社会关系）
2. 确保有制衡机制（强者有弱点，弱者有价值）
3. 标注独特卖点
4. 指出潜在的"崩坏"风险
5. 使用模板格式输出
```

### 3.2 势力设计

```
请阅读 _guides/01_world_guide.md 中关于势力设计的部分。

帮我设计 [数量] 个主要势力，基于以下世界观：
[你的世界观概述]

要求：
1. 势力之间有张力/冲突
2. 符合Wayne原则W3（避免绝对忠诚）
3. 每个势力都有内部矛盾
4. 使用模板格式输出
```

### 3.3 世界观一致性检查

```
请阅读以下世界观设定文件：
- 01_world/power_system.md
- 01_world/rules.md
- 01_world/factions.md

检查是否有：
1. 内部矛盾/逻辑漏洞
2. 违反Wayne原则的设定
3. 可能导致后期"崩坏"的问题

如有问题，请提供修改建议。
```

## 四、人物阶段 (Phase 3)

### 4.1 主角设计

```
请阅读以下文件：
- _guides/02_characters_guide.md（人物设计指南）
- 00_meta/concept.md（创意概念）
- 00_meta/guardrails.md（创作红线）

帮我设计主角，基于以下初步想法：
[你的主角想法]

要求：
1. 使用主角人物卡模板
2. 设计三层动机（表层/深层/核心）
3. 选择合适的人物模板
4. 设计多面性（公开面/私密面/隐藏面）
5. 设计弧光（起点→转折→终点）
6. 使用B20检查清单评分
```

### 4.2 配角设计

```
请阅读以下文件：
- _guides/02_characters_guide.md（人物设计指南）
- _INDEX.md（现有人物）
- 00_meta/guardrails.md（创作红线）

帮我设计一个配角，定位是：
[配角功能/与主角的关系]

要求：
1. 使用配角人物卡模板
2. 重点检查Wayne原则R1（非依附关系）
3. 确保有独立于主角的个人目标
4. 明确"会反对主角的情况"
5. 明确"可能离开主角的情况"
```

### 4.3 关系网络设计

```
请阅读以下人物卡：
[列出已设计的人物]

帮我设计这些人物之间的关系网络：
1. 绘制关系矩阵
2. 确保有足够的张力
3. 检查是否有"养狗"关系
4. 为每对关系设计发展轨迹
```

### 4.4 人物质量评估

```
请阅读以下人物卡：
[人物卡内容]

使用B20检查清单评分：
1. 内核评分（核心矛盾清晰度）
2. 动机评分（三层动机完整度）
3. 多面性评分（多个面向）
4. 一致性评分（行为符合人设）
5. 独特性评分（记忆点）
6. 弧光评分（成长/变化）

总分：___/60

并给出具体改进建议。
```

## 五、剧情阶段 (Phase 4)

### 5.1 主线设计

```
请阅读以下文件：
- _guides/03_plot_guide.md（剧情设计指南）
- 00_meta/concept.md（创意概念）
- 02_characters/c001_主角.md（主角设计）

帮我设计主线弧光：
1. 使用主线弧光模板
2. 确保与主角弧光对应
3. 标注关键转折点
4. 识别潜在风险点
```

### 5.2 支线设计

```
请阅读以下文件：
- 03_plot/main_arc.md（主线）
- _INDEX.md（人物/势力索引）

帮我设计 [数量] 条支线：
1. 使用支线模板
2. 明确与主线的关系（直接推动/间接铺垫/人物成长/世界观展开/节奏调节）
3. 确保类型多样（人物线/势力线/谜团线）
```

### 5.3 伏笔规划

```
请阅读：
- 03_plot/main_arc.md（主线）
- 03_plot/subplots.md（支线）

帮我规划伏笔系统：
1. 为每条主要剧情线设计1-2个伏笔
2. 包含短期(5-20章)/中期(20-100章)/长期(100章+)伏笔
3. 使用伏笔登记簿模板
4. 确保每个伏笔都有回收计划
```

## 六、大纲阶段 (Phase 5)

### 6.1 整体结构规划

```
请阅读以下文件：
- 00_meta/concept.md（创意概念）
- 03_plot/main_arc.md（主线弧光）

帮我规划整体结构：
1. 预计总体量
2. 卷划分（每卷主题、核心冲突、主角成长）
3. 整体节奏规划
```

### 6.2 黄金三章设计

```
请阅读以下文件：
- 00_meta/concept.md（创意概念）
- 02_characters/c001_主角.md（主角设计）
- 01_world/power_system.md（力量体系）

帮我设计黄金三章：
1. 每章详细大纲（使用单章大纲模板）
2. 确保有足够的钩子
3. 确保读者在3章内理解：主角是谁、想要什么、阻碍是什么
4. 标注情绪曲线
```

### 6.3 分章大纲生成

```
请阅读以下文件：
- 04_outline/vol_01/summary.md（第一卷概要）
- 03_plot/main_arc.md（主线弧光）
- 03_plot/foreshadowing.md（伏笔规划）

帮我生成第 [X-Y] 章的详细大纲：
1. 使用单章大纲模板
2. 控制节奏张弛
3. 标注伏笔埋设/回收
4. 确保每章有"下章钩子"
```

### 6.4 节奏检查

```
请阅读 04_outline/vol_01/chapters.md

检查节奏问题：
1. 情绪曲线是否有张有弛？
2. 是否有连续多章都是"铺垫"？
3. 高潮点分布是否合理？
4. 提供具体的节奏调整建议
```

## 七、写作阶段 (Phase 6)

### 7.1 写作前Context加载

```
请阅读以下文件，准备写第 [X] 章：

1. _STATUS.md - 当前状态
2. _INDEX.md - 人物/伏笔索引（重点看要出场的人物和待处理伏笔）
3. _chapter_log.md - 最近10章摘要
4. 04_outline/vol_01/chapters.md - 第[X]章大纲

确认你理解了：
- 当前剧情进展
- 本章目标
- 出场人物的性格特点
- 需要埋设/回收的伏笔
```

### 7.2 章节写作

```
基于上述context，请写第 [X] 章。

要求：
1. 字数约 [XXXX] 字
2. 遵循大纲但可以合理发挥
3. 注意对话区分度
4. 章末留下钩子
5. 标注本章伏笔操作（如有）
```

### 7.3 写作后更新

```
第 [X] 章写作完成。请帮我更新：

1. _chapter_log.md - 添加本章摘要
   格式：章节|标题|主要事件|出场人物|伏笔操作|字数

2. _INDEX.md - 如有新人物/新伏笔/新事件，添加条目

3. 03_plot/foreshadowing.md - 如有伏笔操作，更新状态
```

### 7.4 10章Wayne原则自检

```
请阅读第 [X-Y] 章内容，进行Wayne原则自检：

使用以下模板检查：

## 第 X-Y 章 Wayne原则自检

### R1 "养狗"检查
- 配角是否有自己的行动？
- 配角是否有反对/质疑主角的时刻？

### R2 符号化检查
- 配角是否只展示了单一面向？
- 是否有角色可以用一个词概括？

### R3 草菅人命检查
- 是否有无意义的死亡？
- 死亡是否有余波？

### P1-P4 核心原则检查
- 是否尊重了每个出场角色？
- 配角是否有自己的想法？
- 强弱对话是否有人格平等？

给出具体的问题章节和改进建议。
```

### 7.5 一致性检查

```
请阅读：
- _INDEX.md（人物索引）
- 第 [X-Y] 章内容

检查人物一致性：
1. 人物行为是否符合人设？
2. 人物语言是否有区分度？
3. 人物关系是否有发展？
4. 有无明显的设定冲突？
```

## 八、通用提示词

### 8.1 快速状态确认

```
请阅读 _STATUS.md 和 _INDEX.md，告诉我：
1. 当前进度
2. 主要待办事项
3. 可能存在的问题
```

### 8.2 完整项目回顾

```
请阅读以下核心文件：
- _STATUS.md
- _INDEX.md
- 00_meta/concept.md
- 03_plot/main_arc.md
- 05_draft/_chapter_log.md

给我一个项目状态总结：
1. 当前完成情况
2. 主线进展
3. 人物发展
4. 伏笔状态
5. 建议的下一步
```

### 8.3 问题诊断

```
我在 [具体环节] 遇到了问题：
[描述问题]

请帮我分析：
1. 问题根源
2. 可能的解决方案
3. 需要修改哪些文件
```
