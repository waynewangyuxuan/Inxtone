## 一、主线弧光设计

### 1.1 主线结构

主线应有清晰的三幕结构变体：

| 阶段 | 功能 | 占比 | 关键任务 |
|------|------|------|----------|
| 开篇 | 建立、吸引 | 10-15% | 黄金三章、钩子 |
| 上升 | 发展、铺垫 | 30-40% | 小高潮、势力登场 |
| 中点 | 转折、危机 | 10-15% | 重大转折 |
| 下降 | 困境、准备 | 20-30% | 最大危机、资源集结 |
| 高潮 | 决战、解决 | 10-15% | 主线收束 |
| 尾声 | 收尾、伏笔 | 5-10% | 余韵、下一卷伏笔 |

### 1.2 主线弧光与主角弧光的对应

主线应与主角成长同步：

| 主线阶段 | 主角弧光对应 |
|----------|--------------|
| 开篇 | 主角起点状态展示 |
| 上升 | 主角初步成长、小胜利 |
| 中点 | 主角认知转变 |
| 下降 | 主角面对致命弱点 |
| 高潮 | 主角完成转变/做出最终选择 |
| 尾声 | 主角新状态确立 |

## 二、支线设计

### 2.1 支线类型

| 类型 | 功能 | 示例 |
|------|------|------|
| **人物线** | 丰满配角 | 配角的身世揭秘 |
| **势力线** | 世界观展开 | 宗门内斗、国家阴谋 |
| **谜团线** | 悬念维持 | 神秘遗迹、未解之谜 |
| **感情线** | 情感共鸣 | 爱情、友情发展 |
| **过渡线** | 节奏调节 | 轻松日常、修炼时光 |

### 2.2 支线与主线的关系

支线应服务于主线，有以下几种关系：

| 关系 | 描述 | 示例 |
|------|------|------|
| **直接推动** | 支线完成推动主线进展 | 获得关键道具 |
| **间接铺垫** | 支线为主线做铺垫 | 揭示反派弱点 |
| **人物成长** | 支线让主角/配角成长 | 学会新技能 |
| **世界观展开** | 支线展现世界的其他面 | 了解势力格局 |
| **节奏调节** | 支线调节阅读节奏 | 紧张后的轻松 |

### 2.3 支线模板

```markdown
## [ID] [支线名称]

### 概述
- **类型**: [人物线/势力线/谜团线/感情线/过渡线]
- **状态**: [⬜未启动 / 🔄进行中 / ✅已完结 / ❌废弃]
- **优先级**: [高/中/低]

### 内容
- **核心冲突**:
- **关联人物**: [C00X, C00X]
- **关联势力**: [F00X]

### 与主线关系
- **关系类型**: [直接推动/间接铺垫/人物成长/世界观展开/节奏调节]
- **具体作用**: [这条支线如何服务主线？]

### 计划
- **启动章节**: Ch.XX
- **关键节点**:
  - Ch.XX: [事件]
  - Ch.XX: [事件]
- **收束章节**: Ch.XX
- **收束方式**: [如何结束？开放/闭合？]

### 人物影响
- **主角影响**: [这条支线如何影响主角？]
- **配角影响**: [关联配角有什么变化？]
```

## 三、伏笔系统

### 3.1 伏笔类型

| 类型 | 周期 | 作用 |
|------|------|------|
| **短期伏笔** | 5-20章 | 小悬念、即时满足 |
| **中期伏笔** | 20-100章 | 阶段性揭秘 |
| **长期伏笔** | 100章+ | 贯穿全书的谜团 |

### 3.2 伏笔设计原则

1. **埋设时要自然**: 不能太刻意
2. **回收时要惊喜**: 读者应该"啊，原来如此"
3. **有计划地管理**: 不能埋了就忘
4. **适度即可**: 不必每个细节都是伏笔

### 3.3 伏笔登记簿模板

```markdown
# 伏笔登记簿

## 待回收

### [FS001] [伏笔名称]

#### 埋设
- **章节**: Ch.XX
- **内容**: [埋设了什么？]
- **原文**: "[引用原文，方便回收时对照]"

#### 真相
- **实际含义**: [这个伏笔背后的真相是什么？]

#### 回收计划
- **计划章节**: Ch.XX
- **回收方式**: [如何揭示？]
- **预期效果**: [读者应该有什么反应？]

#### 关联
- **关联人物**: [C00X]
- **关联剧情**: [P00X/S00X]

---

### [FS002] ...

---

## 已回收

### [FS00X] [伏笔名称]
- **埋设**: Ch.XX
- **回收**: Ch.XX
- **效果**: [实际效果如何？]

---

## 放弃

### [FS00X] [伏笔名称]
- **原因**: [为什么放弃？]
- **处理**: [是否需要处理？]
```

## 四、死亡美学（来自B20第三部分）

当剧情涉及角色死亡时，遵循Wayne原则R3（禁止草菅人命）：

### 4.1 死亡设计检查

| 问题 | 期望回答 |
|------|----------|
| 如果这个死亡发生在主角身上，你会这样写吗？ | 会 |
| 这个角色死后，有人会真正悲伤吗？ | 有 |
| 这个死亡除了"证明反派残忍"还有什么作用？ | 有其他作用 |

### 4.2 有意义的死亡设计

| 类型 | 特征 | 示例 |
|------|------|------|
| **牺牲型** | 为他人/理想而死 | 师父护徒 |
| **完成型** | 完成使命后死去 | 复仇者的最后一击 |
| **转折型** | 死亡改变其他角色 | 好友之死促使主角觉醒 |
| **悲剧型** | 命运使然的悲剧 | 无法避免的结局 |

### 4.3 死亡的余波

死亡不应是终点，而应产生余波：

- **情感余波**: 其他角色的悲伤、愤怒、成长
- **剧情余波**: 因死亡而改变的格局
- **主题余波**: 死亡承载的主题意义
