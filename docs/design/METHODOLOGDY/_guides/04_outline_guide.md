## 一、整体结构规划

### 1.1 卷的划分原则

| 维度 | 建议 |
|------|------|
| 单卷字数 | 30-60万字（根据平台和类型） |
| 单卷章节 | 100-200章（每章2000-3000字） |
| 每卷目标 | 一个完整的小闭环 |

## 二、黄金三章设计

### 2.1 黄金三章的重要性

根据FRAMEWORK_V3.0，前三章决定读者是否继续阅读。

| 章节 | 必须完成的任务 |
|------|----------------|
| 第1章 | 钩子、主角登场、冲突萌芽 |
| 第2章 | 冲突升级、世界观暗示、第二个钩子 |
| 第3章 | 明确方向、读者锁定 |

### 2.2 黄金三章检查清单

- [ ] 第1章有足够的钩子（读者想知道"然后呢"）
- [ ] 主角在前3章内登场
- [ ] 主要冲突在前3章内建立
- [ ] 读者能理解：主角是谁、想要什么、阻碍是什么
- [ ] 有至少一个让读者"想继续看"的悬念

### 2.3 常见钩子类型

| 类型 | 描述 | 示例 |
|------|------|------|
| **悬念钩** | 制造疑问 | "三年前他死过一次" |
| **冲突钩** | 即时冲突 | 开篇即遇敌 |
| **反转钩** | 打破预期 | 以为弱鸡其实隐藏高手 |
| **情感钩** | 引发共鸣 | 师父临终托付 |
| **世界钩** | 独特世界观 | 诡异的力量体系 |

## 三、分章大纲格式

### 3.1 大纲详略原则

| 范围 | 详略程度 | 说明 |
|------|----------|------|
| 前10章 | 详细 | 每章300-500字大纲 |
| 11-30章 | 中等 | 每章100-200字大纲 |
| 31章+ | 简略 | 可以只写关键节点 |

### 3.2 单章大纲模板

```markdown
### Ch.XX [章节标题]

**本章目标**: [这章要完成什么？]

**场景**:
- 场景1: [地点] - [事件] - [结果]
- 场景2: [地点] - [事件] - [结果]

**出场人物**: [C001, C002, ...]

**情绪曲线**: [低→高 / 高→低 / 平稳]

**伏笔操作**:
- 埋设: [FS00X]
- 回收: [FS00X]

**下章钩子**: [读者看完这章想继续看的理由]
```

## 四、节奏控制

### 4.1 情绪曲线设计

好的小说应该有张有弛：

```
高 |    /\      /\
  |   /  \    /  \/\
  |  /    \  /
  | /      \/
低 |________________
   开篇  上升 中点 下降 高潮
```

### 4.2 节奏检查

| 章节范围 | 节奏要求 |
|----------|----------|
| 前3章 | 必须有钩子，不能太慢 |
| 前10章 | 至少1个小高潮 |
| 每20-30章 | 至少1个高潮点 |
| 卷末 | 大高潮 + 悬念 |

### 4.3 FRAMEWORK_V3.0节奏维度

根据FRAMEWORK_V3.0的建议：

| 节奏类型 | 适合情况 | 风险 |
|----------|----------|------|
| 快节奏 | 追求爽感、新书期 | 容易疲劳 |
| 慢热型 | 世界观复杂、人物驱动 | 新书期流失 |
| 变奏型 | 兼顾深度和节奏 | 需要精细控制 |

## 五、第一卷小闭环

### 5.1 什么是小闭环

第一卷结束时，读者应该有：
- **满足感**: 某个问题得到解答/解决
- **期待感**: 新的更大问题出现

### 5.2 小闭环设计

```markdown
## 第一卷闭环

### 解决的问题
[第一卷开头提出的问题，在卷末得到什么程度的解决？]

### 留下的问题
[什么更大的问题留到下一卷？]

### 主角状态变化
[主角从什么状态变成什么状态？]

### 格局变化
[世界/势力格局有什么变化？]
```
